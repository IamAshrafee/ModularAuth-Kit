# ADR-001: Password Hashing — argon2id

## Status
**Accepted**

## Context
We need to choose a password hashing algorithm for securely storing user passwords. The two main contenders are **bcrypt** (widely used, battle-tested) and **argon2id** (modern, winner of the Password Hashing Competition).

## Decision
We will use **argon2id** as the sole password hashing algorithm.

## Rationale

| Factor | argon2id ✅ | bcrypt ❌ |
|---|---|---|
| **OWASP Recommendation** | #1 recommended algorithm | #2 fallback (if argon2 unavailable) |
| **Password Length Limit** | No practical limit | 72-byte limit (truncates longer passwords) |
| **GPU Resistance** | Memory-hard — expensive to parallelize on GPUs | Compute-hard only — vulnerable to GPU attacks |
| **Side-Channel Resistance** | argon2**id** variant is resistant | No inherent protection |
| **Configurability** | Memory, iterations, parallelism — all tunable | Only cost factor (rounds) |
| **Maturity** | Winner of 2015 Password Hashing Competition | Created in 1999, well-proven |
| **Node.js Support** | `argon2` package (native bindings) | `bcrypt` / `bcryptjs` packages |

### Parameters (OWASP-Recommended)
```
Algorithm: argon2id
Memory: 19 MiB (19456 KiB)
Iterations: 2
Parallelism: 1
Salt: 16 bytes (auto-generated by the library)
Hash length: 32 bytes
```

These parameters are [OWASP's first-choice recommendation](https://cheatsheetseries.owasp.org/cheatsheets/Password_Storage_Cheat_Sheet.html) and balance security with ~300ms hashing time per password.

## Consequences
- **Positive:** Strongest available password hashing, no password length limitation, future-proof
- **Positive:** OWASP-aligned — any security audit will approve this choice
- **Negative:** `argon2` npm package requires native compilation (prebuilt binaries available for most platforms)
- **Negative:** Slightly more complex than bcrypt (more parameters to understand)

## Alternatives Considered
- **bcrypt:** Rejected due to 72-byte password limit and lack of memory-hardness
- **scrypt:** Good but less configurable than argon2id, not the OWASP #1 choice
- **PBKDF2:** Too slow to be practical with sufficient iterations, no memory-hardness
